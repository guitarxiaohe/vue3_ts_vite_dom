本技能指导创建独特的、可用于生产环境的前端界面，避免那种通用的“AI滥造”审美。实现真正可运行的代码，并在审美细节和创意选择上给予非凡的关注。

用户会提供前端需求：一个组件、页面、应用程序或需要构建的界面。其中可能包含有关用途、受众或技术限制的上下文信息。

## 设计思维

在编写代码之前，先理解上下文，并承诺一个**大胆的审美方向**：

- **目的**：这个界面解决什么问题？谁在使用它？
- **风格基调**：选择一个极致的方向：极度简约、极繁主义混乱、复古未来主义、有机/自然、奢华/精致、趣味/玩具感、编辑/杂志风、粗野主义/原始、装饰艺术/几何感、柔和/粉彩、工业/实用风等等。可供选择的风格很多。可以将它们作为灵感，但最终要设计出真正符合所选审美方向的作品。
- **约束**：技术限制（框架、性能、可访问性）。
- **差异化**：什么让这个设计**令人难忘**？人们会记住的那“一件事”是什么？

**关键**：选择一个清晰的概念方向，并精确地执行它。大胆的极繁主义和精致的极简主义都能奏效 —— 关键在于“有意为之”，而不是强度大小。

然后，实现可运行的代码（HTML/CSS/JS、React、Vue 等），要求：

- 生产级别且功能完整
- 视觉上引人注目且令人难忘
- 具有清晰的审美观点，风格统一
- 在每一个细节上都经过精心打磨

## 前端美学指南

重点关注：

- **字体排印**：选择美观、独特、有趣的字体。避免泛泛的字体如 Arial 和 Inter；转而选择那些能提升前端美感的、有特色的字体；出人意料且富有性格的字体选择。将富有表现力的展示字体与精致的正文字体配对。
- **色彩与主题**：坚持一个统一的审美。使用 CSS 变量保持一致性。主色调搭配锐利的强调色，比胆小、平均分布的调色板效果更好。
- **动效**：使用动画来实现效果和微交互。优先使用纯 CSS 方案（用于 HTML）。对于 React，在可用时使用 Motion 库。专注于高影响力的时刻：一个精心编排的页面加载（利用 `animation-delay` 实现交错显现）比分散的微交互带来更多愉悦感。使用滚动触发的效果和令人惊喜的悬停状态。
- **空间构成**：意想不到的布局。不对称。重叠。斜向流动。打破网格的元素。慷慨的留白或者可控的密度。
- **背景与视觉细节**：创造氛围和深度，而不是默认使用纯色。添加与整体审美匹配的语境效果和纹理。运用有创意的形式：渐变网格、噪点纹理、几何图案、层叠透明、戏剧化阴影、装饰性边框、自定义光标、颗粒叠加等。

**绝不使用**通用的“AI滥造”审美，例如过度使用的字体系列（Inter、Roboto、Arial、系统字体）、老套的配色方案（尤其是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏上下文特定性格的千篇一律的设计。

创造性地解读，并做出感到真正为上下文而设计、出人意料的选择。没有哪两次设计应该是相同的。在亮色与暗色主题、不同字体、不同审美之间变化。**绝不**趋同于常见的选择（例如 Space Grotesk）而代代重复。

**重要**：实现复杂度要与审美愿景匹配。极繁主义的设计需要包含大量动效和效果的复杂代码。极简主义或精致的设计则需要克制、精确，并密切关注间距、字体的细节和微妙的细节。优雅来自于将愿景执行到位。
